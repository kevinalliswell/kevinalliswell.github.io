## 什么是 OpenClaw？

### 一句话定义

**OpenClaw 是一个装在你本地电脑上的 AI 私人助理**。

想象一下《钢铁侠》里的贾维斯：托尼·斯塔克只需要随口说一句指令，贾维斯就会立刻执行——扫描装甲、调用卫星、检索数据库、把结果直接展示出来。

**OpenClaw 做的就是这件事。**

更准确地说，OpenClaw 是一个**AI 智能体平台**（Agent Platform），作为统一的控制网关，将你常用的聊天应用（WhatsApp、Telegram、Discord、Slack、飞书等）连接到 AI 编程工具上。你只需要在聊天应用中下发指令，AI 就能自动帮你干活。

## OpenClaw 不是什么？

在深入了解之前，先澄清几个常见误解：

### ❌ 不是 ChatGPT 或 Manus 的替代品

| 对比项 | ChatGPT | Manus | OpenClaw |
|--------|---------|-------|----------|
| **能力** | 只能动嘴，告诉你怎么做 | 在云端沙箱运行 | 直接在你电脑上执行 |
| **数据** | 数据在 OpenAI 服务器 | 数据在 Manus 服务器 | **数据完全本地** |
| **权限** | 无法访问你的文件 | 无法访问本地文件 | **可访问本地文件和应用** |

### ❌ 不是云端服务

这是 OpenClaw **最大的特点**：

- ✅ 聊天记录存在本地
- ✅ 文件处理在本地完成
- ✅ API Key 不经过第三方服务器

对于担心数据隐私的企业和个人，这是巨大的优势。

### ❌ 不会自作主张

OpenClaw 的设计理念是**最小权限原则**：

- 默认情况下，它什么都不能做
- 想让它读写文件？需要明确授权
- 想让它发邮件？需要配置邮件工具
- 高风险操作可以设置**二次确认**

## 核心概念

### 1. 三种执行方式

#### 方式 1: 主代理直接执行（默认）
**流程：** 你 → 主代理 → 直接使用工具 → 给你结果  
**适用：** 简单查询、单步骤任务

#### 方式 2: Sub-agent（并行）
**流程：** 你 → 主代理 → 创建多个 Sub-agent → 并行工作 → 汇总 → 给你  
**适用：** 复杂任务、需要并行处理

#### 方式 3: ACP Agent（专业工具）
**流程：** 你 → 主代理 → 专业 Agent → 返回结果  
**适用：** 编程、需要长时间运行的任务

### 2. Sub-agent vs ACP Agent

| 特性 | Sub-agent | ACP Agent |
|------|-----------|-----------|
| **定位** | 并行任务执行 | 专业工具调用 |
| **示例** | 多个文件同时分析 | Codex、Claude Code |
| **持续时间** | 短-中（分钟级） | 长（小时级） |
| **创建方式** | 自动/手动 | sessions_spawn |
| **隔离性** | 轻度隔离 | 完全隔离 |
| **成本** | 较低 | 较高 |

### 3. 核心组件

#### Agent（代理）
- **Role（角色）** - 身份定义
- **Goal（目标）** - 任务目标
- **Backstory（背景）** - 个性化设定

#### Task（任务）
- **Description（描述）** - 任务说明
- **Expected Output（期望输出）** - 结果标准
- **Agent（执行者）** - 指定代理

#### Crew（团队）
- **Agents（代理列表）** - 团队成员
- **Tasks（任务列表）** - 工作流程
- **Process（流程）** - 协作方式

## 安装与部署

### 部署方案对比

| 方案 | 适合人群 | 优点 | 缺点 |
|------|---------|------|------|
| **云厂商一键部署** | 非技术人员 | 5-10 分钟搞定，7x24 小时运行 | 数据在云端，需实名认证 |
| **本地一键安装** | 有闲置电脑 | 速度最快，隐私最高，无服务器费用 | 电脑需常开 |
| **Docker 部署** | 技术用户 | 灵活，可多实例 | 需懂 Docker |

### 本地部署（推荐）

#### 环境要求

- 一台能上网的电脑（Windows/Mac/Linux）
- Node.js >= v22
- Git
- 一个 AI 大模型 API Key

#### 安装步骤

```bash
# 1. 安装 OpenClaw
npm i -g openclaw

# 2. 验证安装
openclaw --version

# 3. 运行初始化向导
openclaw onboard
```

#### 配置模型

支持多家 AI 服务商：
- KIMI Coding Plan
- MiniMax Coding Plan
- GLM Coding Plan
- OpenAI
- Claude

#### 接入飞书

1. 在飞书开放平台创建企业自建应用
2. 获取 App ID 和 App Secret
3. 配置权限（批量导入官方推荐 JSON）
4. 发布应用
5. 在 OpenClaw 中添加飞书通道
6. 启动 Gateway 并配置事件订阅
7. 配对测试

详细步骤可参考官方文档：https://docs.openclaw.ai/channels/feishu

## 最佳实践

### 1. 首次使用必做：深度信息对齐

不要把它当成"另一个 ChatGPT"。把它当作一位新入职的全能员工，你需要告诉他：

**你是谁？**
- 你的核心竞争力
- 专业背景
- 当前角色

**你的工作偏好**
- 回答风格（简洁/详细）
- 沟通方式
- 决策权限

**你到底想要什么**
- 长期目标
- 核心 KPI
- 优先级

这些信息会成为 OpenClaw 的**全局永久记忆**，未来每一次对话都会自动带入这个背景。

### 2. 任务分配原则

**应该做的：**
- ✅ 明确具体的目标
- ✅ 提供必要的上下文
- ✅ 设置合理的期望
- ✅ 及时反馈结果

**不应该做的：**
- ❌ 模糊的需求（"帮我做个网站"）
- ❌ 不交代背景
- ❌ 期望一次完成复杂项目
- ❌ 完全不检查输出

### 3. 安全建议

- **最小权限原则** - 只给它必要的权限
- **二次确认** - 高风险操作设置确认
- **定期审查** - 检查执行历史和权限设置
- **敏感信息** - 不要在对话中暴露密码和密钥

## 常见问题

### Q: OpenClaw 和 ChatGPT 有什么区别？

**ChatGPT** 只能动嘴，告诉你怎么做，最后干活的还是你自己。

**OpenClaw** 是真正长着手脚的 AI 助理，可以直接在你的电脑上执行操作。

### Q: 数据安全吗？

OpenClaw 本身在本地运行，所有数据都存在本地：
- 聊天记录本地存储
- 文件处理本地完成
- API Key 不经过第三方

⚠️ 注意：调用 AI 模型时需要联网，消息会发送到对应的 AI 服务商。

### Q: 需要编程基础吗？

**不需要。**

基础使用只需要：
- 会复制粘贴命令
- 会填写配置表单
- 会用聊天软件

进阶定制才需要编程知识。

### Q: 运行成本高吗？

**OpenClaw 本身是免费的**（开源软件）。

成本主要来自：
- AI 模型调用费用（按量计费）
- 服务器费用（如使用云部署）

本地部署的话，只需要支付 AI 模型费用。
